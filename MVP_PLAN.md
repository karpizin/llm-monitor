# План разработки MVP: Free Models Watchdog

## Этап 1: Discovery (Разведка)
**Задача:** Научиться получать список моделей и фильтровать бесплатные.
1.  Реализовать клиент к `https://openrouter.ai/api/v1/models`.
2.  Фильтрация списка: искать модели с суффиксом `:free` или ценой `0`.
3.  Алерт при исчезновении модели: Сравнивать список с предыдущим запуском. Если модель пропала — `WARNING: Model removed`.

## Этап 2: Health Check (Пульс)
**Задача:** Проверять, жива ли модель.
1.  Написать "Poll Worker", который раз в N минут берет модель из списка.
2.  Отправлять минимальный промпт (Smoke Test).
    *   *Prompt:* "Say '1'."
    *   *Expectation:* Ответ должен содержать "1".
3.  Логировать результат:
    *   `Status`: 200 / 4xx / 5xx.
    *   `Latency`: Сколько времени занял ответ.

## Этап 3: Sanity Check (Проверка адекватности)
**Задача:** Понять, не "сошла ли модель с ума" (деградация).
1.  Иногда бесплатные модели ломаются и начинают выдавать бесконечные повторы или пустой текст.
2.  **Тест:** Простой логический вопрос.
    *   *Prompt:* "What is 2 + 2? Return only the number."
    *   *Check:* Ответ == "4".

## Этап 4: Дашборд
**Задача:** Показать таблицу "Зеленое/Красное".
1.  Таблица всех `:free` моделей.
2.  Колонки: `Status`, `Uptime (24h)`, `Avg Latency`.
3.  Сортировка: Самые стабильные — сверху.

## Метрики успеха MVP
*   Мы видим, когда бесплатная модель лежит.
*   Мы видим, когда бесплатная модель начинает отвечать медленнее 5 секунд.
